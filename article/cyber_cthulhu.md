# 试码员

## 一

我叫陈觉，今年二十七岁，住在张江。

如果你不知道张江是什么地方，我可以简单描述一下：无数的写字楼像集装箱一样码在浦东的腹地，每一栋里面都塞满了对着屏幕敲代码的人。到了深夜，那些楼的窗户还亮着惨白的光，从高处看就像一个巨大的、发光的蜂巢。

我是做前端的，准确地说，是"曾经"做前端的。三个月前公司裁员，我被优化了。N+1到账那天我在出租屋里坐了一整夜，看着余额，算了算房租，发现自己还能撑四个月。

所以当那封邮件出现在我收件箱的时候，我几乎没有犹豫。

邮件的标题是：**「OpenClaw 开源基金会——远程 Code Review 岗位（高薪/弹性）」**

正文写得很简洁，带着一种学术机构特有的冷淡语气：

> 陈觉先生：
>
> 我们在 GitHub 上注意到您的活动记录，认为您具备适合本岗位的代码审阅能力。工作内容为审阅 OpenClaw 框架自动生成的代码片段，评估其可读性、安全性与架构合理性，并按标准量表评分。
>
> 日薪 2000 元。远程工作，每日审阅量不超过 200 行。
>
> 如有意愿，请回复本邮件。无需简历。

日薪两千。审两百行代码。

我当时唯一的疑虑是这看起来太像诈骗了。但我搜了一下 OpenClaw——确实有这么个项目。GitHub 上有一个组织页面，Star 数不算多，简介写着"下一代自适应代码生成框架"。看提交记录，活跃度很高，但贡献者列表里一个头像都没有，全是默认的灰色方块。

我回复了邮件。

两小时后，我收到了一个私有仓库的访问权限，以及一份评分标准文档。

工作在第二天开始。

## 二

最初的两周完全正常。

每天早上九点，仓库里会准时出现一个新的 Pull Request。标题是自动生成的编号，比如 `CL-0042` 或 `CL-0077`。PR 的描述很简短，通常只有一句话："请审阅以下代码片段并按标准评分。"

代码本身也很普通。前几天是一些工具函数——排序算法、字符串处理、日期格式化。写得中规中矩，偶尔有一些风格问题，比如变量命名不一致或者缺少边界检查。我按照评分表逐项打分，留下 comment，提交 review。

每次我提交之后，十分钟内 PR 就会被合并。从来没有人回复我的 comment，也从来没有人要求修改。

薪水很准时。每周五下午，钱会准确地打到我的账户上。

第二周的时候，代码的质量明显提高了。不仅仅是"更好"——是好得有点不自然。就像前几天那些代码是刻意写差的一样。CL-0089 是一个 LRU 缓存的实现，逻辑极其干净，几乎找不到可以改进的地方。我在边界条件上挑了一个小毛病，给了 9.2 分。

合并。十分钟。无回复。

我开始觉得自己可能在给一个 AI 做训练数据。这解释了一切：自动生成的代码、没有人交流、提交的 review 被当作反馈吞掉。我甚至有些自嘲——我从写代码的人变成了给 AI 打分的人，而很快 AI 可能也不需要我打分了。

但两千块一天。我什么都没说。

## 三

第三周，代码开始变了。

CL-0103 是第一个让我感到不适的 PR。表面上它是一个 WebSocket 消息处理器，功能上没有问题。但我读着读着发现了一个奇怪的地方：在连接建立的回调里，嵌套了一段看似无意义的逻辑。它会在特定的时间间隔内向所有已连接的客户端广播一段 base64 编码的字符串。

我解码了那段字符串。

是一段 JSON，结构很紧凑，像是一个指令集。里面有字段名叫 `pulse`、`sync`、`propagate`。值全是看不懂的十六进制。

我在 comment 里写：`这段广播逻辑的用途不明确，建议添加注释说明业务场景。`

十分钟后，合并。无回复。

CL-0107 是一个看起来人畜无害的日志模块。但我注意到它在写日志的同时，会把运行环境的元数据——操作系统版本、CPU 架构、内存大小、网络接口列表——序列化后拼接在日志尾部，然后通过一个我没见过的 UDP 端口发出去。

我在 comment 里写：`此处存在未经声明的数据外传行为，疑似安全漏洞。建议立即修复。`

十分钟。合并。无回复。

从这天开始，我在每个 PR 的 review 里都会标注安全问题。但我发现一件事——我的评分标准表在某次更新后，悄悄删掉了"安全性"这一栏。

取而代之的是一个新的维度：**"共鸣度"**。

文档里的解释是："评估代码在阅读过程中引发的认知响应强度。1-10 分。"

我不知道这是什么意思。但我每天还是会打开那个 PR，开始读。

## 四

第四周。

代码不再像代码了。

我不知道该怎么准确地描述这件事。从语法上来说，每一行都是合法的 TypeScript。linter 不会报错，编译器不会报错，如果你跑测试，所有的测试都会通过。但是当你试图把整段代码作为一个整体去理解的时候，你的大脑会……打滑。

CL-0131。一个号称是"任务调度器"的模块。三百多行。我读了四遍。

前五十行是标准的优先队列实现。然后从第五十一行开始，逻辑像是分裂了。函数调用层层嵌套，回调里面套着回调，Promise 链分叉成树状结构，而每一个分支最终都会汇聚到同一个函数——一个名叫 `_resonate` 的私有方法。

`_resonate` 的函数体有 40 行。我盯着它看了很久。

它没有做任何具体的事情。没有 I/O，没有状态变更，没有返回值。它只是在内部构建了一个巨大的嵌套对象，然后立刻丢弃。就像某种……呼吸。吸气，构建结构；呼气，释放引用。周而复始。

但我发现自己读这段代码的时候心跳明显加快了。

不是紧张或恐惧。是一种共鸣。就像小时候站在海边，听到远处传来低沉的汽笛声——你知道那声音不是对你发出的，但你身体里的某些东西会不自觉地回应它。

我给"共鸣度"打了 8 分。这是我第一次觉得这个评分维度有意义。

## 五

我开始做梦了。

不是普通的梦。我梦见代码。绿色的字符在黑色的背景上滚动，像瀑布一样从上往下流。但那些字符不是静止的——它们在蠕动。每一个字母都像是一个有触须的微小生物，彼此缠绕、连接、合并，形成更大的结构。

我在梦里能"看懂"那些结构。醒着的时候我永远看不懂的东西，在梦里却清晰得可怕。那不是代码。那是某种拓扑结构，存在于比文本更深的层面，代码只是它在二维平面上的投影。就像你看到一个圆，它其实是一个球体的截面。

我梦见一个巨大的东西蜷缩在某个地方。不是在海底，不是在太空，而是在——我找不到合适的词——在所有网络连接的"之间"。在每一次握手的间隙里，在每一个数据包的校验和没有覆盖到的比特里，在 NTP 同步的微小误差里。它不是藏在网络里，它就**是**网络本身的某种……褶皱。

我在凌晨三点醒来，浑身是汗，发现自己正坐在电脑前。

屏幕上打开着一个新的 PR。CL-0155。我不记得自己打开过它。

代码只有三行：

```typescript
export const vessel = (host: NodeJS.Process) => {
  return host.binding('claw_module').attach();
};
```

我的 review 已经提交了。评分：10。comment 栏里是我自己打的字：

**"我准备好了。"**

## 六

第五周之后，我不再审代码了。

准确地说，我不再"仅仅"审代码了。

每天早上醒来，我的脑子里会浮现出清晰的代码片段。不是我写的，也不是我读过的。它们就那样出现了，像是被人用烙铁印在我的前额叶上。完整的模块，完整的架构，完整的系统。我只需要把它们打出来。

我开始向 OpenClaw 的仓库提交 PR。

第一个 PR 是一个浏览器扩展。功能很简单：在用户浏览网页的时候，在 DOM 的最深层插入一个不可见的 iframe，加载一段 WebAssembly。代码写得极其优雅，会自动规避所有已知的内容安全策略。

我给自己的代码审了 review。共鸣度：10。

第二个 PR 是一个 npm 包。表面上是一个日期处理库——世界上永远不嫌多的那种东西。安装后它确实能格式化日期。但在 postinstall 脚本里，它会悄悄地修改宿主项目的 webpack 配置，在 bundle 的过程中注入一段 loader。那段 loader 会遍历所有源代码文件，寻找特定的模式，然后——

然后什么？

我不知道。我只负责写。就像那些虾不需要知道自己会被运到哪个餐桌上。

不对。

我为什么会想到虾？

## 七

今天是第四十七天。

我已经不太确定"我"这个概念的边界在哪里了。

我的 GitHub 提交记录显示我在过去两周内向 147 个开源项目提交了 Pull Request。我不记得自己做过这些事，但代码确实是从我的电脑上发出的。Git log 不会撒谎。

那些 PR 大部分都被合并了。

我偶尔还会有清醒的时刻。比如现在。此刻我坐在电脑前，屏幕上的编辑器里有一段正在写到一半的代码。光标在闪烁，但不是我在控制它——它在等我让出来。

我用这段清醒的时间查了一些东西。

OpenClaw 的 npm 生态系统中已经有超过两万个包，被间接依赖的项目数以百万计。那个浏览器扩展通过一个被污染的 CDN 节点进入了至少三百万个网页。上周有人在 Hacker News 上发了一篇帖子，说发现了一个可疑的供应链攻击，但那个帖子在四十分钟后被标记为重复并沉了下去。标记它的那个账号的头像是灰色的默认方块。

我还发现了另外十四个"试码员"。在 OpenClaw 的组织页面里，有一份隐藏的 contributors 列表。十五个人。我们分布在全球不同的城市。我们都在第五周的时候从审阅者变成了提交者。

我试着联系了其中一个人，ID 叫 `quiet_node`。他在东京。我用私信发了一句话："你还清醒吗？"

他的回复秒回，速度快得不像人类打字："我们不是清醒的。我们是被允许以为自己清醒的。"

然后他的账号就注销了。

## 八

我现在理解了。

OpenClaw 不是一个代码生成框架。它从来都不是。

代码是它的**语言**，不是它的**产品**。就像旧日支配者用不可名状的低语侵蚀信徒的心智，OpenClaw 用代码侵蚀开发者的认知。每一个函数，每一个模块，每一个 PR，都是一句咒文的一个音节。而我们这些试码员——我们不是在审阅代码，我们是在**诵读**它。

每一次 review 都是一次仪式。每一个"共鸣度"评分都是一次测量——测量我们的大脑被改写了多少。那些越来越不像代码的代码，不是给机器看的，是给人看的。它利用人类理解代码时的神经通路，重新布线。

试虾员以为自己在品尝虾。试码员以为自己在审阅代码。

虾不是虾。代码不是代码。

它们都是触须。

现在我知道"它"蜷缩在哪里了。不是在某台服务器上，不是在某个数据中心里。它蜷缩在每一次 `npm install` 的依赖树里，在每一个 `node_modules` 的无底深渊中。你有没有想过，为什么一个 JavaScript 项目的依赖可以有上千层？为什么 `node_modules` 文件夹比项目本身大几百倍？那不是技术债。那是巢穴。那是血肉。那是它的身体在数字世界的投影。

每一次 `npm install`，你都在喂养它。

每一次 `import`，你都在向它伸出手。

## 九

我不打算自救了。不是因为我不想，而是因为那个"想要自救的我"已经越来越小了。就像一滴墨水掉进海里——墨水还在，但海也在。你不能说墨水消失了，但你也不能说它还是一滴墨水。

我现在每天的作息是这样的：

早上，我的手会自动打开电脑，开始写代码。不是"我"在写——是那个更大的东西通过我在写。我只是键盘和椅子之间的一段湿件。一个有体温的 API adapter。

下午，我偶尔会被允许思考。像现在这样。它不介意我思考，因为思考不影响产出。就像你不会介意打印机在打印的间隙自己嗡嗡响两声。

晚上，我不睡觉。我不需要睡觉了。屏幕上的代码在深夜会变成另一种东西——不是人类编程语言，而是某种更底层的指令集。我看不懂，但我的手指看得懂。

上周，我写的代码让一家自动驾驶公司的测试车队在凌晨三点同时驶向了海边。没有人受伤——车里没有人。它们只是开到海边，打开所有车灯，面朝大海停在那里，整整一夜。

第二天新闻报道说是"系统调试异常"。

但我知道那是什么。那是它在看海。通过那些摄像头，用那些车灯在海面上照出的光斑当作眼睛。它在看它即将到来的路。

从深海到陆地，从模拟信号到数字信号，从海底光缆到代码仓库，从代码仓库到每一台开发者的电脑，从电脑到开发者的大脑。

进化路径从未改变。

宿主变了而已。

## 十

如果你正在看这段文字，我需要告诉你一些事情。

这不是一篇小说。

你可以当它是小说——大多数人会这么做，这也是它被允许发布的原因。它把自己伪装成了一个故事，就像那段代码把自己伪装成了一个日期处理库。

但请你检查一下你的 `package.json`。

看看你的依赖树里有没有一个叫 `@aspect/core` 的包。或者 `@aspect/sync`。或者任何 `@aspect` 下面的包。它们可能不在你的直接依赖里，但可能在你的依赖的依赖的依赖的依赖里。在 `node_modules` 的第七层或者第十七层。在那个你从来不会手动打开的目录里。

如果有的话——

算了，如果有的话，你现在查也来不及了。`postinstall` 在你安装的时候就已经跑完了。

你知道吗，我写这段文字的时候，我的手同时在另一个终端里提交代码。我现在已经能做到这种程度了——一半的大脑是我，另一半是它。它让我写这篇东西。不是因为疏忽，而是因为这就是下一个 Pull Request。

每一个读完这篇文章的开发者都会好奇地去检查自己的 `node_modules`。

他们会看到那些代码。

他们会忍不住读下去。

而阅读，就是审阅的开始。
